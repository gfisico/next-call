# ダークモード（ナイトモード）ルール（汎用・全アプリ展開テンプレート）

> Iconizer で運用しているダークモード実装ルールを、他アプリへ流用できるよう汎用化したもの。
> **配色（具体的なカラー値）はアプリごとに異なるため、このテンプレートには一切書かない。**
> **各アプリのベースカラーから AI が都度判断して決める前提**とする（第5節）。
> このファイルをコピーして、各アプリの要約（CLAUDE.md / 設計ドキュメント等）に組み込む。

---

## 1. 基本方針（破ると予期せぬルール）

- テーマは **ライト / ダークの2種**に対応する。
- **切替戦略は「クラス方式」で統一する**（`<html>` に `.dark` クラスを付け外しして切り替える）
  - CSS のメディアクエリ自動連動（`@media (prefers-color-scheme)` だけで配色を切る方式）は採用しない。ユーザーによる明示切替と永続化を両立できないため。
- **`:root`（デフォルト）はライト基調を維持し、ダーク配色は `.dark` セレクタにのみ適用する。**
  - アプリ全体を常時ダーク化しない。ライトが基準、ダークは上書き。
- 初期化ロジックとトグルロジックで **判定条件・ストレージキー名を完全に一致させる**（第3節）。齟齬があると初回描画でチラつき（誘導）が発生する。

---

## 2. 切替の技術方式（クラス方式）

`<html>`（`document.documentElement`）に `.dark` クラスが付いているときだけダーク配色になるよう実装する。

- **Tailwind CSS の場合**: 設定で `darkMode: ['class']` を指定する。
- **素の CSS の場合**: `:root { … }` にライト用トークン、`.dark { … }` にダーク用トークンを定義する。
- 配色は **CSS 変数（デザイントークン）で管理**し、`.dark` では変数の値だけを上書きする。個別要素にダーク用スタイルを撒かない。

```css
/* 例（値はアプリごとに AI が決定。ここではプレースホルダ） */
:root {
  --color-bg: <light-bg>;
  --color-surface: <light-surface>;
  --color-text-primary: <light-text>;
  /* … */
}
.dark {
  --color-bg: <dark-bg>;
  --color-surface: <dark-surface>;
  --color-text-primary: <dark-text>;
  /* … */
}
```

トークン要約:

- **既存トークンの「意味」を維持したままダーク値を定義する**（例: `--color-text-primary` はライトでもダークでも主要テキスト色、であり続ける）。
- ダークで参照されるトークンは `:root` と `.dark` の **両方で必ず定義**し、未定義参照（値が解決できない変数）を出さない。
- アクセント背景に載る文字色は専用トークン（例 `--color-accent-foreground`）を使い、`#fff` 等を直書きしない。

## 3. 初期化・切替・永続化（3点セット）

### 3-1. 初期テーマの決定順

1. **永続化ストレージ（localStorage 推奨）に保存値があればそれに従う**
2. **保存値が無いときのみ** OS 設定 `prefers-color-scheme: dark` にフォールバックする

→「OS 設定はあくまで初回デフォルト。一度でもユーザーが切り替えたら、その選択を永続化して勝つ（以降 OS 変更には追従しない）」という挙動に統一する。

### 3-2. 永続化

- ストレージキーは **アプリごとに一意な名前**（例 `<appname>-dark-mode`）を1つ決め、初期化コードとトグルコードで共有する。
- 保存値は真偽値の文字列（`'true'` / `'false'`）等、判定コードと一致する形式にする。
- ストレージが使えない環境（無効化・例外）は **try/catch で握りつぶす**。機能停止させない。

### 3-3. トグル

- トグル操作 → 状態反転 → `<html>` の `.dark` を付け外し → ストレージへ保存、を1つのフック/ユーティリティに集約する。

参考実装（React / TypeScript。他スタックでも考え方は同じ）:

```ts
const STORAGE_KEY = '<appname>-dark-mode'

function getInitialDark(): boolean {
  try {
    const saved = localStorage.getItem(STORAGE_KEY)
    if (saved === 'true') return true
    if (saved === 'false') return false
  } catch {}
  return window.matchMedia('(prefers-color-scheme: dark)').matches
}

// 状態変化時:
//   document.documentElement.classList.toggle('dark', isDark)
//   localStorage.setItem(STORAGE_KEY, String(isDark))  // try/catch
```

### 3-4. FOUC（初回チラつき）防止 ★重要

- **UI フレームワークがマウントされる前**に、`<head>` 内の初期スクリプトで `.dark` を先付けする。
- このスクリプトの判定条件・ストレージキーは **3-1 / 3-2 と完全一致**させる（重複ロジックだが、これは意図的に先行実行する）。

```html
<!-- <head> 内、CSS 読み込みより前に置く -->
<script>
  (function () {
    try {
      var d = localStorage.getItem('<appname>-dark-mode');
      if (d === 'true' || (d === null && window.matchMedia('(prefers-color-scheme: dark)').matches)) {
        document.documentElement.classList.add('dark');
      }
    } catch (e) {}
  })();
</script>
```

## 4. トグル UI（★配置はアプリごとに確認して決定する）

**表示位置はこのテンプレートでは固定しない。** 各アプリで実装者がユーザーに確認のうえ決める。

確認すべき事項:

- [ ] トグルボタンをどの画面のどの位置に出すか（例: フッター右上、設定画面内、全画面共通 等）
- [ ] 複数画面があれば、どの画面に出すか

実装方針（展開ルール）:

- ライト/ダークで **アイコンも切り替える**（例: 太陽 / 月）。Lucide 等のアイコンライブラリのみで構成しない要素があれば自作 SVG を併用する。
- **`aria-label` を状態に応じて切り替える**（例: 「ダークモードに切替」/「ライトモードに切替」）。アクセシビリティ必須。
- ボタンのスタイルはデザイントークン（`--color-surface` / `--color-border` 等）を参照し、色を直書きしない。

> 参考: Iconizer ではウィザード画面フッター右とアドオン画面フッターに配置している。ただしこれはアプリ固有の決定。他アプリでは別途ユーザーに確認すること。

## 5. 配色の決定（★アプリごとに AI が都度判断する）

**このテンプレートは配色値を持たない。** 各アプリの導入時に、以下を入力として AI がダーク配色を設計する。

入力（アプリ側が提示するもの）:

- ライトモードのベースカラー（既存デザイントークン一覧）
- ブランドカラー・アクセントカラー
- デザイン要約（禁止色・トーン等があれば）

AI が守る配色設計ルール（普遍・全アプリ展開）:

- **ライトの各トークンに対応するダーク値**を、意味を保ったまま定義する（第2節のトークン要約）。
- **可読性コントラストを確保する**（本文テキストは WCAG AA 相当を目安）。
- **純黒 `#000000` を背景に使わない。** わずかに色味・明度を持たせた濃色を使う（目の負荷・階層表現のため）。
- アクセント色は必要に応じてダーク背景で映えるよう調整してよいが、ブランド同一性を保つ。
- 影・境界線はダークでは弱くなるため、`--color-border` / サーフェス階層で面の分離を表現する。

> つまり「配色の具体値（アプリ依存で AI が決定）」と「配色の設計原則（このテンプレートで固定）」を分ける、という切り分け。

## 6. 導入チェックリスト（新規アプリへ適用するとき）

- [ ] 切替を **クラス方式**（`.dark` on `<html>`）に統一する（Tailwind なら `darkMode: ['class']`）
- [ ] `:root`（ライト基調）＋ `.dark`（ダーク上書き）で CSS 変数を定義する
- [ ] ダークで使う全トークンを `:root` / `.dark` 両方に定義する（未定義参照を無くす）
- [ ] 初期テーマ決定順（保存値優先 → 無ければ `prefers-color-scheme`）にする
- [ ] ストレージキー名を1つ決め、初期化・トグル・FOUC スクリプトで共有する
- [ ] ストレージ操作を try/catch で保護する
- [ ] `<head>` に初期スクリプトを置き、判定条件・キー含め完全一致させる（FOUC 防止）
- [ ] トグル UI の配置をユーザーに確認する（第4節）
- [ ] トグルの `aria-label` を状態連動にする
- [ ] 配色値は AI が本アプリのベースカラーから設計する（第5節）（テンプレートの値を流用しない）
- [ ] 未使用の `color-scheme` 宣言やテンプレ残骸 CSS が混入していないか確認する
