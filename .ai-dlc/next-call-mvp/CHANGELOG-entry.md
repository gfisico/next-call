# next-call MVP — Changelog

## [0.1.0] — 2026-07-13

ジャズセッションで「次に何をコールするか」を、当日の演奏・過去の履歴・編成・意図から提案するアプリの初期リリース。

### Added

- **セッション記録**: 店舗選択（新規登録時のみ母店判定）・リスナー客トグルでセッションを開始し、演奏した曲を追加（曲名検索・クイック登録・参加/楽器・コール有無・譜面なし・フロント編成の順序付き記録）。セットリスト編集・削除・セッション終了・過去セッション閲覧。モバイル片手操作最適化（375px）。
- **推薦エンジン**（DB非依存の純関数パイプライン・9ステージ）: 完全除外5条件、スコアリング（スライダー5・チェック2・減点8種・ジャンル上書き加点）、繰り返し減点、softmax重み付き非復元抽選（seed再現可能）、理由生成、条件別候補（編成不明時の分岐）、保留曲注釈。カバレッジ99%超。
- **推薦API**: DB集計から EngineInput を組み立て、推薦履歴を永続化、選曲意図の前回値引き継ぎ、保留曲のコール時自動解除。曲500・演奏5,000件で p95 < 2秒。
- **選曲支援画面**: 編成条件・制約・意図（iOS風5段階スライダー）を1画面で調整し、候補約3曲を推奨理由付きで表示。条件別候補・保留曲枠・「この曲をコール」導線。
- **マスタ・設定画面**: 曲マスター一覧/属性編集（needs_review補完）、エンジン設定調整、CSVインポートウィザード（4段階）、全データエクスポート。
- **一括インポート**: songs.csv / setlists.csv による曲マスター＋約5年分履歴の投入（プレビュー→解決→ドライラン→コミットの単一トランザクション）、および初回限定の Excel 抽出スクリプト（CLI）。
- **認証**: Google OAuth（許可メールのみ）。未認証は全ルートで /login へリダイレクト。
- **インフラ**: multi-stage Dockerfile（standalone・起動時マイグレーション）、docker compose（app + Caddy 自動TLS）、GitHub Actions CI/CD（quality→image→deploy）、週次バックアップ20世代＋ピン留めスナップショット、運用ドキュメント（Xserver VPS）。
